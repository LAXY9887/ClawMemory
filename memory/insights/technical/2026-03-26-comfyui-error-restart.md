# ComfyUI 報錯處理流程

## LXYA 指定原則
> "以後遇到 ComfyUI 報錯先重啟再說"

## 執行邏輯
1. **不分析** - 報錯時不立即深入分析
2. **先重啟** - 重啟 ComfyUI Desktop 應用
3. **再判斷** - 重啟後問題是否解決

## 適用理由
- 多數報錯是暫時性狀態問題
- 重啟成本低，效果明顯
- 避免過度分析浪費時間
