# 2026-03-28 - Stock Pipeline 穩定性革命

## 🛠️ 關鍵技術修復

### ComfyUI Timeout 機制實施
- **txt2img**: 180秒 timeout (3分鐘) 
- **img2img**: 180秒 timeout (3分鐘)
- **upscale**: 300秒 timeout (5分鐘，因upscale較慢)
- **實現**: ThreadPoolExecutor + future.result(timeout)

### 異常處理革命
- **批次隔離**: 單張圖片失敗不影響其他圖片
- **重試保護**: 重試過程異常處理
- **詳細記錄**: 異常類型和錯誤信息完整保存

### 根本問題診斷
- **Stage 3-4 卡住原因**: brief-002 在refined階段只生成4/5張，第5張時ComfyUI調用timeout
- **缺失**: 沒有timeout機制導致無限等待
- **影響**: 整個批次處理無法完成

## 🔧 JSON 元數據修復

### 問題發現
- pipeline_upscaled_00010_.json 中提示詞顯示 "test prompt" (測試數據覆蓋真實數據)

### 解決方案
- 重新生成JSON，使用prompt_history.json中的真實提示詞
- **正面提示詞**: "Dawn Fog, Golden Light, Dynamic Motion, Pink Ponytail Outline..." (516字元)
- **完整元數據**: 包含creative_brief、canvas_info、pipeline_metadata

## 🐛 緊急修復

### BOM字符問題
- **症狀**: image_refiner.py 出現語法錯誤 "invalid non-printable character U+FEFF"
- **原因**: 編輯過程中意外引入BOM (Byte Order Mark)
- **解決**: 移除BOM字符，語法檢查通過

## 💡 重要經驗

### Timeout設計原則
- **txt2img/img2img**: 3分鐘足夠大部分生成
- **upscale**: 5分鐘因為處理量更大
- **ThreadPoolExecutor**: 比signal.alarm更穩定的跨平台方案

### 異常處理策略
- **繼續性原則**: 單個失敗不中斷整體流程
- **詳細記錄**: 每種失敗都要有可追蹤的log
- **優雅降級**: 失敗時返回結構化錯誤而非崩潰

### 文件編碼注意
- **BOM危險**: UTF-8 BOM會導致Python語法錯誤
- **檢測方法**: 用ast.parse()驗證語法
- **清理方法**: 二進制模式讀取並移除前3個BOM字節
