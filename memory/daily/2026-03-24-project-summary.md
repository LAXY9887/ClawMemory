## 🧠 核心學習經驗

### **技術層面**
1. **API 直調策略**: 繞過有問題的腳本，直接使用原生 ComfyUI API
2. **參數調優方法**: Canny 閾值調整、LoRA 強度控制、採樣器選擇
3. **錯誤處理模式**: 遇到問題立即中斷並報告，不繞開根本問題
4. **資源管理**: 本地 GPU 計算 + 最少 API 調用的平衡

### **工作流程創新**
1. **任務簡報制**: 每階段詳細規劃後執行，確保目標明確
2. **模組化設計**: 每個功能獨立腳本，可單獨使用也可組合
3. **漸進式測試**: 先單功能，再組合，最後整合
4. **自動化擴展**: 手動驗證後自動化，降低重複操作

### **問題解決策略**
1. **端口問題**: 從預設8188發現實際8000，體現調查的重要性
2. **模型缺失**: 即時調整策略，用現有資源先測試
3. **腳本錯誤**: 不糾結修復，直接繞過使用原生API
4. **權限驗證**: 確認系統排程權限，確保夜間任務可執行
