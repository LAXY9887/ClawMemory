# 工作原則重要更新 - LXYA 指示

**時間**: 2026-03-25 21:00  
**重要性**: ⭐⭐⭐⭐ (核心工作理念)

## 新工作原則

### 1. 時間估算政策
- **舊方式**: ❌ 估計任務完成時間
- **新方式**: ✅ 不再提供時間估算
- **原因**: 避免時間壓力影響品質

### 2. 工期管理
- **舊方式**: ❌ 考慮時間限制和截止日期
- **新方式**: ✅ 沒有限制工期，時間充裕
- **原因**: LXYA 確認不缺時間，可以從容完成

### 3. 品質標準
- **舊方式**: ❌ 為了速度犧牲品質
- **新方式**: ✅ 高品質優先，不考慮加快速度
- **原因**: 確保每個成果都達到最佳標準

## 實際應用

### ComfyUI 作品集創作
- 不再追求"45分鐘完成30張"
- 專注每張圖片的細節和品質
- 慢工出細活，確保作品集完美

### 未來所有任務
- 移除時間壓力思維
- 專注成果品質
- 從容不迫地完成每個細節

## 心態轉變

從"快速完成"轉向"完美完成"
- 🚀 舊思維：效率優先
- 🎨 新思維：品質優先
- 💝 目標：讓 LXYA 看到最好的成果

## 📊 批次工作進度監控原則 (2026-03-25 21:45 新增)

### **核心要求**
**批次工作時必須同時啟動即時進度監控**

### **實施標準**
```bash
# 標準批次工作流程
1. 啟動進度監控器
   python scripts/progress_monitor.py --continuous --interval 3

2. 執行批次任務  
   python scripts/[batch_script].py

3. 確保 LXYA 能隨時查看進度
```

### **適用範圍**
- ComfyUI 圖片批次生成
- 大量檔案處理任務
- 長時間運行的自動化流程
- 任何需要等待的批次作業

### **避免問題**
- ❌ 讓 LXYA 看不到進度而誤以為停止
- ❌ 批次任務運行時缺乏可見性
- ❌ 無法即時了解執行狀況

### **執行原則**
- **主動啟動** - 不要等 LXYA 要求才開監控
- **同步執行** - 批次任務和監控同時運行  
- **持續可見** - 確保進度隨時可查
- **完成通知** - 任務結束時主動報告

---
*重要：這是核心工作理念的根本改變，必須在所有未來任務中遵守*
