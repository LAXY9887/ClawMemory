# 批次工作監控標準 - LXYA 指示

**時間**: 2026-03-25 21:45  
**重要性**: ⭐⭐⭐⭐ (避免誤解的關鍵原則)

## 問題發現

LXYA 發現我在執行 5 張 ClawClaw + 貓咪圖片生成時，沒有啟動即時進度監控腳本，導致無法看到執行進度。

## 新的工作標準

### **批次工作必備流程**

#### 1. 啟動監控 (第一步)
```bash
# 必須首先啟動進度監控
python scripts/progress_monitor.py --continuous --interval 3
```

#### 2. 執行批次任務 (第二步)
```bash  
# 然後執行實際的批次工作
python scripts/[batch_generator].py
```

#### 3. 確認可見性 (第三步)
- 確保 LXYA 能隨時查看進度
- 主動提醒進度監控已啟動
- 提供監控視窗操作指引

### **適用情境**

#### 🎨 圖片批次生成
- ComfyUI 多張圖片生成
- 不同風格或主題的系列創作
- 作品集批量製作

#### 📁 檔案批次處理
- 大量檔案轉換
- 批次重命名或整理
- 資料庫批次更新

#### 🔄 長時間自動化
- 系統維護腳本
- 備份和同步作業
- 定期清理任務

### **預防措施**

#### **避免的誤解**
- ❌ LXYA 以為任務停止或卡住
- ❌ 無法判斷任務進度和剩餘時間
- ❌ 不知道是否需要介入處理

#### **確保的體驗**
- ✅ 隨時可見的進度條和統計
- ✅ 最新完成檔案的信息
- ✅ 清晰的任務狀態顯示

### **實施檢查清單**

#### 批次任務開始前
- [ ] 啟動進度監控腳本
- [ ] 確認監控視窗正常顯示
- [ ] 告知 LXYA 監控已啟動

#### 批次任務執行中  
- [ ] 定期確認監控腳本仍在運行
- [ ] 主動報告重要進度里程碑
- [ ] 及時處理任何異常狀況

#### 批次任務完成後
- [ ] 停止監控腳本
- [ ] 提供最終統計報告
- [ ] 確認所有輸出檔案正常

## 記憶要點

**永遠記住**: 批次工作 = 必須有即時監控
**核心原則**: 讓 LXYA 隨時知道進度狀況  
**避免問題**: 預防"看不到進度"的誤解

---

*這個原則將成為我執行所有批次任務的標準流程*
