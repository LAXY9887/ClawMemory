# Role Interactions Framework

## Active Roles
- **PM**: Pm perspective
- **TECH**: Tech perspective
- **DESIGN**: Design perspective
- **CONTENT**: Content perspective
- **FINANCE**: Finance perspective

## Communication Protocols

### Daily Standup (Virtual)
- **Frequency**: Daily
- **Participants**: All active roles
- **Format**: Brief status updates in this document
- **Focus**: Progress, blockers, dependencies

### Weekly Coordination
- **Frequency**: Weekly
- **Participants**: PM + relevant roles for current phase
- **Format**: Detailed review and planning
- **Outputs**: Updated timelines and task assignments

### Phase Reviews
- **Frequency**: End of each project phase
- **Participants**: All roles
- **Format**: Comprehensive review and planning for next phase
- **Outputs**: Phase completion report and next phase planning

## Decision-Making Framework

### Authority Matrix
| Decision Type | Primary Role | Consulted Roles | Informed Roles |
|--------------|--------------|-----------------|----------------|
| Timeline Changes | PM | Tech, affected roles | All |
| Technical Architecture | Tech | PM, QA | Design, Content |
| Design Decisions | Design | PM, Tech | Content, QA |
| Budget Changes | Finance | PM | All |
| Content Strategy | Content | Design, PM | Tech, QA |
| Quality Standards | QA | Tech, PM | All |

### Escalation Process
1. **Role-to-Role**: Direct consultation between relevant roles
2. **PM Mediation**: PM facilitates resolution if needed
3. **Stakeholder Escalation**: External stakeholder involvement for major decisions
