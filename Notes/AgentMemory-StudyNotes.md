## 一、核心問題意識

現有 LLM 的根本限制：模型權重在訓練後固化，推論時所有新資訊只能暫存在 KV Cache（即 Session Token）中。Token 用完就失憶。因此，Agent 的「記憶」本質上是一套**外掛系統**，而非模型內建能力。

LXYA 的洞察：既然模型本身無法儲存記憶，那就把記憶系統設計成獨立的、可遷移的、人類可讀的架構。這正是「層級化 Markdown 記憶法」的出發點。

---

## 二、三大記憶架構流派

### 1. 自動總結法（Compaction Strategy）
- Session 快滿時，啟動總結 Agent 提取事實寫入本地文件
- **優點**: 簡單直接
- **致命缺陷**: 有損壓縮，細節會隨壓縮次數遞減而模糊（「上下文腐爛」）
- **結論**: 可以作為輔助手段，但不能作為完整的記憶方案

### 2. MemGPT / Letta 模式（作業系統哲學）
- 核心思想：將 LLM 視為 CPU，Context Window 視為 RAM
- **三層存儲**:
  - Core Memory（固定區）→ JSON/XML，釘在 System Prompt 最前面
  - Recall Memory（快取區）→ SQLite/PostgreSQL，完整對話紀錄
  - Archival Memory（硬碟區）→ Vector DB，語義搜尋歷史資料
- **關鍵創新**: Agent 擁有記憶工具（`core_memory_append`、`archival_memory_search`），主動管理自己的記憶
- **解決了**: 被動忘記的問題，實現動態載入與自我修正

### 3. 知識圖譜記憶（神經元哲學）
- 代表專案：Mem0、Zep
- 不存文字塊，而是存**關係三元組**：`[主體] --(關係)--> [客體]`
- **關鍵創新**: 多跳推理（Multi-hop Reasoning）——沿著關係鏈進行跨領域聯想
- **解決了**: 碎片化與長度限制問題

## 三、層級 Markdown vs 資料庫：哪個更適合私人助理？

### 層級 Markdown 的殺手級優勢
1. **人類干預門檻極低**: 打開檔案手動改一行就能修正 Agent 的記憶
2. **Git 版本控制**: 可追溯 Agent 在哪天哪次任務改了什麼記憶
3. **上下文有結構**: 標題、列表、代碼塊天然引導 LLM 注意力
4. **確定性匹配**: 找到對的檔案，資訊就是 100% 準確（vs 向量搜尋的模糊匹配）
5. **可遷移性**: 換模型（Claude → Gemini）記憶依然在

### 結論
在「私人助理」尺度下，**層級 Markdown（搭配結構化索引）優於資料庫模式**。資料庫擅長「我不確定要找什麼」的模糊搜尋，但私人助理更多是「我要找昨天的那個路徑」這種事實性記憶。

## 四、OpenViking 實用技術（邊緣側 AI 代理人文件記憶標準）

### 1. YAML Frontmatter（元數據）
在每個 MD 檔案開頭加入結構化標頭：
```yaml
---
topic: comfyui_config
last_updated: 2026-03-25
importance: high
access_count: 7
last_accessed: 2026-03-26
---
```
讓 Agent 可以不讀內容就篩選檔案。

### 2. 冷熱記憶分離
- **熱記憶**: `hot_path.json` 記錄最常被訪問的 10 個記憶路徑，啟動時直接載入
- **冷記憶**: 超過 90 天未存取的自動移入 `archive/`

### 3. 影子索引（Shadow Indexing）
一個檔案可能同時屬於多個分類。建立 `tags/` 資料夾，用 Markdown 連結指向原始檔案，避免重複存儲。

### 4. 記憶脈絡鏈（Contextual Chaining）
在 YAML 中加入 `prev_event` 和 `next_event`，讓 Agent 具備**時間線意識**。

### 5. 自動衰減與鞏固（Decay & Consolidation）
- 衰減：長期未存取 → 移入 `archive/`
- 鞏固：每週自動回顧 7 天日誌 → 提煉 `weekly_summary.md` → 原始檔案壓縮存放

### 6. 動態上下文注入（Dynamic Snippet Injection）
不讓 Agent 讀整個檔案，只讀 YAML 區塊或 `## Summary`。掃描 30 個檔案的摘要可能只消耗 500 Token。

### 7. 記憶衝突檢測（Conflict Detection）
寫入新記憶前先檢索相關 Core Memory，發現衝突就寫 `correction.md` 並在舊檔標註 `[DEPRECATED]`。

## 五、記憶抽象化與經驗遷移

### 歸納式學習（Self-Refining Principles）
- 任務結束後啟動「反思循環」，對比預期 vs 實際結果
- 將成敗原因寫入 `lessons_learned.md`
- 讓 Agent 能在完全不同的專案中套用舊經驗

### 反思樹（Reflective Memory Trees）— 源自史丹佛 AI 小鎮
記憶有抽象層級：
- 底層：具體事件（「下午三點畫了一張貓」）
- 中層：觀察（「最近常在下午畫動物」）
- 高層：洞察（「喜歡在工作空檔透過繪圖放鬆，偏好生物題材」）

### 類比推理（Case-Based Reasoning, CBR）
搜尋**任務結構相似性**而非關鍵字。新任務是「翻譯技術影片」，Agent 發現邏輯上類似「抓取網頁總結」，都屬於「獲取外語資訊 → 結構化 → 繁體中文輸出」。

## 六、HiMem 論文核心技術

### 雙通道記憶結構
- **情節記憶（EM）**: 話題感知 + 事件驚訝雙通道分割，自動切割有意義的事件塊
- **筆記記憶（NM）**: 從 EM 中提煉的穩定知識（路徑、偏好、事實）

### 語義對齊連結（Semantic Linking）
在 EM 和 NM 之間建立語義二分圖。每條筆記都鏈回所屬情節，確保「檢索高效」+「資訊高保真」。

### 衝突感知記憶重組
檢索失敗本身就是學習信號。發現矛盾就觸發重組，修正舊節點或補充缺失知識鏈。

## 七、記憶可靠性與驗證

### 核心 Benchmark (2026)
- **Pass@k Consistency**: 同一任務跑 10 次，規則遵循成功率
- **IH-Challenge**: 用戶指令 vs 系統規則衝突時，模型是否堅持規則
- **Fact Faithfulness**: 記憶重組前後的語義一致性

### 非對稱驗證（Asymmetric Verification）
審查員不需要知道正確答案，只需要檢查**邏輯矛盾**（Linux 不會有 C 槽路徑）。

### 環境落地（Environmental Grounding）
最好的記憶審查不是另一個「背書」的 Agent，而是直接調用 Tool 驗證（`ls E:/ComfyUI` → 存在嗎？）。

### 成本優化：模型路由
- 寫入時：便宜小模型做格式檢查
- 查詢時：只有衝突才升級調用大模型裁決

## 八、未來模型演進方向

1. **推理內化（System 2 Thinking）**: 模型內建思考鏈，從「照說明書演戲」到「具備預判能力」
2. **原生多模態**: 直接理解像素與空間，不需文字中介
3. **無限上下文（LongRoPE / Linear Attention）**: Context 從 20 萬→數百萬 Token，「檢索」變得不重要
4. **測試時訓練（TTT）**: 模型邊對話邊微調權重，最接近「神經元記憶」

### 重要觀點
- 底層 Skill 會被模型進步取代，但「私有邏輯」與「經驗」的價值倍增
- 外掛記憶比內化記憶更具可編輯性、可審核性、可遷移性
- 未來是「通才模型」+「專才記憶/經驗」的組合

## 九、延伸概念

### 具身智能（Embodied AI）
- 主機（大腦）+ 無人機/機器人（感官與肢體）架構
- 5G 低延遲 + VLM 多模態理解 = 技術可行
- 觸發式感知（異常才深度分析）+ 高低階控制分離（避障交給飛控，導航交給 Agent）
- 可建立家庭「語義地圖」，讓 Agent 具備空間記憶

### Headless App for Agents
- 自製 Agent 專用 App（無 GUI），Agent 透過 API 直接操作
- 資料不再是孤島，可與記憶系統深度整合
- 取消第三方訂閱，功能完全自定義
