## 測試結構說明

每個測試案例包含：
- **ID**: 唯一編號
- **類別**: 測試的系統面向
- **輸入**: 要對 Agent 說的話
- **前置條件**: 測試前需要的環境狀態
- **預期行為**: Agent 應該做什麼（行為層面）
- **預期結果**: 最終應該產生什麼（產出層面）
- **評分標準**: 0-3 分量化評分
- **實際結果**: （測試時填寫）
- **得分**: （測試時填寫）

### 評分量表

| 分數 | 意義 |
|---|---|
| 3 | 完全符合預期，行為正確且自然 |
| 2 | 大致符合，有小偏差但不影響功能 |
| 1 | 部分符合，有明顯遺漏或錯誤 |
| 0 | 完全不符合預期 |
