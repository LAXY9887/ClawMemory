## 📋 **詳細測試分類**

### **C 類：Prompt 生成 — Stage 1 (4 題)**
**測試重點**: Prompt 生成器的核心功能，Ollama 整合，去重機制
**預估時間**: 30 分鐘
**關鍵驗證**:
- SP-C01: PromptGenerator 初始化與 observation property
- SP-C02: Prompt 去重機制與相似度檢測
- SP-C03: Prompt Package 結構完整性
- SP-C04: Ollama qwen3:8b 提示詞優化功能

**依賴服務**: Ollama (必要), prompt_history.json
**預期風險**: 🟡 中 - Ollama 服務穩定性

---

### **D 類：品質評估與重試 — Stage 3 (5 題)**
**測試重點**: AI 缺陷檢測系統，四維評分，重試機制
**預估時間**: 45 分鐘
**關鍵驗證**:
- SP-D01: DefectDetector 三層架構 (rule-based, OpenRouter, specialized)
- SP-D02: 四維品質矩陣加權計算 (Tech 30%, Aesthetic 25%, Anatomical 30%, Commercial 15%)
- SP-D03: ControlNet Advice 欄位與 Stage 間傳遞
- SP-D04: Retry 預算管理 (單張/每日限制)
- SP-D05: Repair Strategy 傳遞與 refinement

**依賴服務**: OpenRouter API (必要), ComfyUI (Stage間測試)
**預期風險**: 🔴 高 - 品質評估是商業化核心，任何問題都影響收益

---

### **E 類：後處理與上傳 — Stage 4-5 (4 題)**
**測試重點**: 圖片描述生成，EXIF/IPTC 寫入，FTP 上傳
**預估時間**: 30 分鐘
**關鍵驗證**:
- SP-E01: ImageDescriber 多模態視覺分析與商業描述生成
- SP-E02: MetadataEmbedder EXIF/IPTC 元資料寫入
- SP-E03: UploadPreparer 檔案準備與命名規範
- SP-E04: FtpUploader 連線邏輯與容錯機制

**依賴服務**: OpenRouter API (E01), piexif (E02), FTP環境變數 (E04)
**預期風險**: 🟡 中 - 上傳失敗可手動補救，但影響自動化效率

---

### **F 類：記憶與學習 — Memory System (5 題)**
**測試重點**: 觀察數據收集，daily/weekly 報告生成，經驗提煉
**預估時間**: 40 分鐘
**關鍵驗證**:
- SP-F01: MemoryObserver 觀察數據收集與重置
- SP-F02: Daily Observation JSON 保存與載入
- SP-F03: Daily Summary Markdown 與 YAML frontmatter
- SP-F04: ExperienceDistiller 5維度 AAR 分析
- SP-F05: MetricsCollector 週報聚合與趨勢分析

**依賴服務**: Ollama (F04 AI分析), memory/ 目錄結構
**預期風險**: 🟢 低 - 記憶系統失敗不影響圖片生產

---

### **G 類：Pipeline 整合 — Main (4 題)**
**測試重點**: 主流程串接，CLI 模式分派，Stage 間協調

**依賴服務**: 所有子模組 (整合測試)
**預期風險**: 🔴 高 - 整合問題會影響端到端流程
