# Stock Pipeline 素材收集指南

**目標**: 為 LXYA 提供完整的文本庫和審美標準收集指導  
**建立時間**: 2026-03-28  
**預計完成**: 收集完成後可立即啟動 Pipeline 測試  

---

## 📚 1. 文本庫收集任務

### 🎯 **收集目標**
```
目標總量: 4 個主題 × 3-5 篇文章 = 12-20 篇文章
每篇文章: 1000-3000 字，包含豐富的視覺描述詞彙
語言要求: 英文為主 (利於 AI 圖像生成)
```

### 📋 **主題分配**

#### **Tech-Abstract (技術抽象)**
- **關鍵詞**: technology, digital art, cyberpunk, futuristic, sci-fi
- **建議來源**: 科技設計文章、數位藝術教學、未來主義美學分析
- **收集重點**: 材質描述、光效詞彙、科技感元素

#### **Eerie-Scene (神秘場景)**
- **關鍵詞**: mysterious, dark atmosphere, horror, gothic, abandoned
- **建議來源**: 恐怖電影美學分析、哥德式建築、廢墟攝影指南
- **收集重點**: 氛圍營造、陰影效果、神秘元素

#### **ClawClaw-Portrait (人物肖像)**
- **關鍵詞**: character design, anime art, cyberpunk girl, portrait
- **建議來源**: 角色設計教學、動漫風格分析、肖像攝影技巧
- **收集重點**: 人物特徵、表情描述、服裝配件

#### **Creature-Design (生物設計)**
- **關鍵詞**: creature concept, monster design, fantasy beings, biomechanical
- **建議來源**: 概念藝術教學、生物設計分析、奇幻藝術指南
- **收集重點**: 解剖結構、質感描述、奇幻元素

### 💾 **檔案格式**
```
檔名格式: [主題]-article-[編號].txt
範例: tech-abstract-article-1.txt, eerie-scene-article-2.txt

存放位置: stock-pipeline/materials/raw-articles/
```

### 🔧 **自動處理流程**
1. LXYA 將文章放入 raw-articles/ 目錄
2. 執行: `python process_raw_articles.py`
3. ClawClaw 自動分析並更新對應的主題文本庫
4. 檢查生成的 text-library/ 檔案確認結果

## 🎨 2. 審美標準收集任務

### 🎯 **收集目標**
```
目標總量: 4 個主題 × 10-15 張圖片 = 40-60 張參考圖片
圖片要求: 高解析度、風格明確、品質優良
分類要求: good-examples (優秀) + bad-examples (避免)
```

### 📂 **檔案組織**

#### **圖片命名規則**
```
優秀範例: good-[主題]-[編號].jpg
避免範例: bad-[主題]-[編號].jpg

範例:
good-tech-abstract-001.jpg
bad-eerie-scene-002.jpg
```

#### **存放目錄結構**
```
aesthetic-standards/
├── tech-abstract/
│   ├── good-examples/       # 8-12 張優秀範例
│   ├── bad-examples/        # 3-5 張避免範例
│   └── standards.md         # 文字標準 (已建立模板)
├── eerie-scene/            # 同上結構
├── clawclaw-portrait/      # 同上結構
└── creature-design/        # 同上結構
```

### ✅ **優秀範例標準**
- **技術品質**: 高解析度、清晰銳利、色彩準確
- **美學表現**: 構圖平衡、色彩和諧、視覺衝擊力強
- **主題契合**: 風格特徵明顯、符合商業需求
- **商業適用**: 無版權爭議、適合 Stock 平台

### ❌ **避免範例特徵**  
- **技術缺陷**: 模糊、像素化、色彩偏差、曝光問題
- **美學問題**: 構圖失衡、色彩混亂、視覺不適
- **主題偏離**: 風格不符、過於抽象或具象
- **商業風險**: 版權問題、敏感內容、過於小眾

### 📝 **文字標準補充**
基於收集的圖片，LXYA 可以補充 standards.md 中的：
- 具體色彩搭配偏好
- 構圖規則的細節描述  
- 質感和材質的要求
- 避免元素的明確定義

## 🚀 3. 收集完成後的啟動流程

### ✅ **完成檢查清單**
- [ ] 4 個主題的文章收集完成 (12-20 篇)
- [ ] 自動處理腳本執行成功
- [ ] 文本庫檔案內容豐富合理
- [ ] 4 個主題的參考圖片收集完成 (40-60 張)
- [ ] 審美標準文字描述補充完整
- [ ] 目錄結構和檔案命名正確

### 🎯 **啟動準備**
1. **材料驗證**: ClawClaw 檢查所有素材完整性
2. **配置測試**: 確認 Pipeline 能正確讀取素材
3. **小規模測試**: 生成 1-2 張測試圖片驗證流程
4. **正式啟動**: 開始每日 2 張圖片的生產流程

## 💡 收集建議和技巧

### 🔍 **文章來源建議**
- **藝術教學網站**: DeviantArt、ArtStation 教學文章
- **設計部落格**: Creative Bloq、Design Week、Smashing Magazine
- **專業論壇**: ConceptArt.org、CGSociety 討論串
- **學術資源**: 藝術學院的公開課程材料

### 📸 **圖片來源建議**
- **高品質圖庫**: Unsplash、Pixabay 的免費高解析度圖片
- **藝術作品集**: 知名概念藝術家的公開作品
- **商業案例**: 成功的 Stock 圖片作為正面範例
- **教學範例**: 線上課程中的優秀/劣質對比範例

### ⚠️ **注意事項**
- **版權意識**: 僅用於內部參考，不直接使用
- **品質優先**: 寧可數量少一點，也要確保品質
- **多樣性**: 每個主題涵蓋多種子風格和表現形式
- **實用性**: 關注商業 Stock 圖片的實際需求
